# LightFlow v0.2 Checklist

This checklist tracks the work needed to turn the current backend MVP into a
release-ready backend foundation. Keep the scope focused on stable contracts,
runtime verification, and one small UI-enabling slice.

Legend:

- `[x]` verified in this repository.
- `[~]` partially verified; remaining work needs broader runtime or schema
  coverage.
- `[ ]` not complete.

## Release Cleanliness

- [x] `cargo fmt --check` passes.
- [x] `cargo clippy --all-targets -- -D warnings` passes.
- [x] `cargo test` passes with default features.
- [x] Current security and run-history hardening changes are committed.
- [x] `CHANGELOG.md` records user-visible CLI, API, workflow, and runtime
  changes.

## Backend Contract

- [~] OpenAPI paths and schemas match the actual HTTP server behavior.
  `server::tests::openapi_paths_match_http_routes` verifies path parity, and
  `ErrorResponse` documents the shared error shape. Full response-schema
  validation against live endpoint bodies is still pending.
- [x] `lfw info`, `lfw help`, `/nodes`, `/models`, `/runs`, and `/artifacts`
  expose the same workflow/runtime vocabulary through `ApiService`, node-card
  tests, server history tests, and CLI integration tests.
- [x] Path and id segment validation is covered for run history, batch runs,
  patch registry entries, and workflow writes.
- [x] Error responses are structured enough for CLI, MCP, and future editor
  clients. HTTP errors now include `error`, `code`, `message`, and `status`.

## Runtime Verification

- [x] Default preview/mock executors are tested and documented as
  local/demo-safe.
- [~] `--features rig` is tested against at least one mock/local provider path
  with `cargo test --features rig --test llm_rig`. A configured external
  provider smoke test is still pending because it needs credentials or a local
  compatible provider endpoint.
- [x] `LIGHTFLOW_FLUX_RUNNER` is tested with the external runner contract.
- [ ] `--features flux-native` build requirements and supported platforms are
  documented after local verification.
- [x] Model sync and locked-run paths are tested for missing, present, and
  approval-required model artifacts.

## Minimal UI Slice

- [x] `lfw serve` can back a read-only node catalog view using `/nodes`.
- [x] A node detail view can display inputs, outputs, model requirements,
  runtime availability, and validation status from `/nodes/{workflow_id}`.
- [x] A run history view can read `/runs`, `/runs/{run_id}`, and
  `/runs/{run_id}/events`.
- [x] No graph canvas is started until the read-only API-backed slice works.
  `LightFlowUI` remains an intentionally empty placeholder.

## Documentation

- [x] README has one tested quickstart for `lfw init`, `lfw new`, `lfw run`,
  and `lfw serve`.
- [x] `docs/workflow-development.md` has one tested runtime-backed workflow
  example.
- [x] Standard workflow agent skills match their current input/output
  contracts through `lfw node test` coverage in `tests/standard_nodes.rs`.
- [x] Runtime docs clearly label preview, mock, external, native, and reserved
  executor paths.

## Remaining v0.2 Gaps

- Add live response-schema validation for OpenAPI component schemas, not only
  path parity and targeted error-shape tests.
- Run and document a configured external RIG provider smoke test.
- Locally verify `--features flux-native`, then document build prerequisites
  and supported platforms.
