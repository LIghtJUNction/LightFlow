# LightFlow v0.2 Checklist

This checklist tracks the work needed to turn the current backend MVP into a
release-ready backend foundation. Keep the scope focused on stable contracts,
runtime verification, and one small UI-enabling slice.

## Release Cleanliness

- `cargo fmt --check` passes.
- `cargo clippy --all-targets -- -D warnings` passes.
- `cargo test` passes with default features.
- Current security and run-history hardening changes are committed.
- `CHANGELOG.md` records user-visible CLI, API, workflow, and runtime changes.

## Backend Contract

- OpenAPI paths and schemas match the actual HTTP server behavior.
- `lfw info`, `lfw help`, `/nodes`, `/models`, `/runs`, and `/artifacts` expose
  the same workflow/runtime vocabulary.
- Path and id segment validation is covered for run history, batch runs, patch
  registry entries, and workflow writes.
- Error responses are structured enough for CLI, MCP, and future editor clients.

## Runtime Verification

- Default preview/mock executors are tested and documented as local/demo-safe.
- `--features rig` is tested against at least one mock/local provider path and
  one configured external provider path.
- `LIGHTFLOW_FLUX_RUNNER` is tested with the external runner contract.
- `--features flux-native` build requirements and supported platforms are
  documented after local verification.
- Model sync and locked-run paths are tested for missing, present, and
  approval-required model artifacts.

## Minimal UI Slice

- `lfw serve` can back a read-only node catalog view using `/nodes`.
- A node detail view can display inputs, outputs, model requirements, runtime
  availability, and validation status from `/nodes/{workflow_id}`.
- A run history view can read `/runs`, `/runs/{run_id}`, and
  `/runs/{run_id}/events`.
- No graph canvas is started until the read-only API-backed slice works.

## Documentation

- README has one tested quickstart for `lfw init`, `lfw new`, `lfw run`, and
  `lfw serve`.
- `docs/workflow-development.md` has one tested runtime-backed workflow example.
- Standard workflow agent skills match their current input/output contracts.
- Runtime docs clearly label preview, mock, external, native, and reserved
  executor paths.
